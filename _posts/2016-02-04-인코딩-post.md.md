# 2/4 유니코드
 - DB, 사람이 만든 문자, 글자 집합
 - code point, 개념상 숫자

## encoding
 - 그림 : BMP, PNG, JPEG
 - 유니코드
	 - UTF-8 : 공간 효율적, 실제 글자수 알 수 없음,하나의 문자를 3바이트로 표현
	 - UTF-16 : 모든 유니코드 커버 가능, 글자수를 알 수 없지만 배열의 길이는 알 수 있음
	 - UTF-32 : 낭비
	 - UTF-16과 UTF-32는 엔디언에 따라서 글자 내의 바이트 순서가 반대임
 - UCS / UCS2
